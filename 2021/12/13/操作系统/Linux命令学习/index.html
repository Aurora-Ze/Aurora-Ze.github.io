Linux命令学习

发表于 2021-12-13 更新于 2021-12-23 分类于 专业基础 阅读次数：
本文字数： 4.9k 阅读时长 ≈ 4 分钟

一、基本命令

路径

cd [path]：前往指定的路径

  • cd -：返回到上次的路径
  • cd ~：前往当前用户的用户目录
  • cd /：前往根目录

pwd：打印当前路径

目录文件

mkdir [dir_name]：创建一个目录

cp [src] [dst]：复制文件

mv [src] [dst]：移动文件（也可以用来重命名文件）

rm [path]：删除文件

  • -r：表示递归的删除整个目录或文件
  • -f：表示不需要确认

查找文件

1. 搜索命令、程序等

which：在PATH指定的路径中查找。（一般用于查找命令）

whereis：用于程序名的搜索，如二进制、man说明文件和源代码文件

2. 常规搜索

find [path] [condition]：在指定目录中根据某些条件进行查询

  • -name logfile：查询文件名为logfile的文件
  • -perm 644：查询权限为644的文件

locate：可以等价于find -name方式，但是它是从文件数据库中进行查询，因此速度更快。

解压缩

压缩：gzip [path]

解压缩：gzip -d [path]

此外，使用tar -czvf [dst] [src]和tar -zxf [src]也可以实现压缩和解压。

链接

ln [src] [dst]命令可以与文件建立链接，其中：

  • 硬链接：一个文件的所有硬链接使用同一个inode。最开始的源文件可以视为是一个硬链接。
  • 软链接-s：类似于快捷方式的存在，它存储的是源文件的地址，因此源文件删除后就无法再通过软链接来读取

grep与xargs

在管道用法中，比较常见的两个命令就是grep和xargs

  • grep：查询符合条件的字符串

  • xargs：将上游的输出作为下游的输入参数

grep用法

tail -f log | grep "error"

监听并输出log日志，并查询其中的error字符串（即高亮显示）

xargs用法

find . -perm 644 | xargs -I @ mv @ ../newDir

查询当前目录中权限为644的所有文件，并移动到指定目录中

xargs的-I参数可以定义一个指示符，这里定义为**@。然后在后面的mv命令中就可以使用@**来表示参数应放的位置。

二、用户管理

相关命令

操作用户

useradd [username]

passwd [username]

userdel [username]

usermod -g [groupname] [username]：修改某个用户所属的组

操作组

groupadd [groupname]

groupmod [groupname]

groupdel [groupname]

记录

有三个和用户信息相关的文件，分别是：

/etc/passwd：

记录用户基本信息，如用户名、UID、GID等等

/etc/shadow：记录加密形式的用户密码，只有root用户才能够查看

/etc/group：记录组信息

疑惑

测试passwd命令时发现，普通用户可以修改自己的密码，但是/etc/passwd或/etc/shadow对普通用户来说都是没有权限的，那么它是怎么做到的呢？

TIPS

/etc/passwd是文件，存储了用户信息。

/usr/bin/passwd是命令，修改密码时调用的。

答案就在/usr/bin/passwd这个命令自身上。

查看passwd命令的权限，发现一个特殊的权限s：

s(Set UID)表示文件在执行时暂时拥有文件所有者的权限（本例即root权限）。正因为有了root权限，普通用户才得以成功修改密码。

四、Systemd

systemd与systemctl

systemd是Linux系统的一种启动方式，它代替了原先init进程的启动方式。

systemctl是一组控制systemd系统和进行服务管理的命令。

init进程采用串行启动的方式，启动时间较长；而systemd引入了并行启动的概念。

此外，使用systemd后也是存在争议的，因为它违背了Unix的哲学keep it simple, stupid。

systemctl命令

  • systemctl start [service]
  • systemctl stop [service]
  • systemctl restart [service]
  • systemctl reload [service]
  • systemctl status [service]

五、权限控制

传统的权限控制

在Linux系统中，存在一种传统的权限控制方式：UGO，即user-group-other。

ugo把文件的操作者分为三类：所有者、同组用户和其他用户。每类用户都按照读、写、执行来分配权限。

新建的目录权限为775，文件为664，这是由默认的777、666权限减去掩码0002得到的。

现代的权限控制有两种：ACL和SELinux

ACL

Access Control List 是一张表，告知操作系统每个用户对于某个文件来说具备何种权限。

每个文件都拥有一个属性标志着它的权限控制表，表中的每个条目都对应着一个用户所拥有的权限。

ACL是对UGO的扩展，因为UGO进行权限控制的维度是所有者、同组用户和其他用户，无法从更加细化的维度（如每个用户）进行管理。

相关命令

getfacl [file]：打印某个文件的ACL规则

setfacl -m u:[username]:[rwx] [file]：为某个文件设置一条关于用户的ACL规则，权限为rwx

上面提到的两种权限控制都属于自主访问控制（DAC），权限管理的主体是用户，只要用户拥有相应文件的权限，就可以访问该文件。

在DAC中，一旦他人非法获得root权限，那么就会对整个操作系统产生无法预料的影响。

SELinux

SELinux是一种强安全性的权限控制策略，它是强制访问控制（MAC）的一种典型形式。

在SELinux的操作系统中，决定一个资源是否被访问的因素除了上述的读写执行权限外，还需要判断每一类进程是否拥有对某一类资源的访问权限。

六、进程控制

进程

进程是一个正在运行的程序，主要包含PID、PPID、UID、GID和Status等信息。

创建进程

使用fork系统调用，创建一个子进程。除了PID、PPID和一些特殊资源@锁、异步IO、待处理的信号集等外，子进程共享了父进程的其他所有资源@代码、数据和堆栈信息。

实际上，fork采用了写时复制的策略来节省内存拷贝的开销。

在同一段代码中，我们通常利用fork的返回值来区分父子进程：

  • 大于0：在父进程中返回子进程PID
  • 等于0：子进程中返回0
  • 小于0：创建子进程失败

控制进程

信号

信号signals是一种进程级别的中断请求，我们可以用它来控制进程。

  • 查询所有信号kill -l
  • 为某个进程发送信号kill [signals] [pid]

优先级

使用nice -n [number] [name]来设置优先级

监控进程

ps

使用ps命令来查看进程信息。

  • ps -A或ps -e：查看所有进程
  • ps -aux：所有进程的详细信息

top

使用top命令查询进程维度的CPU、内存使用

  • top：查询所有进程的资源使用情况

  • top -p [pid]：查询具体某个进程

strace

跟踪命令或进程的执行情况

  • strace -p [pid]：跟踪进程
  • strace [command]：跟踪命令

lsof

查看进程打开的文件

  • lsof -p [pid]

uptime

查看系统已运行时间和负载情况

前台与后台进程

查看后台执行的命令：jobs(可以得到作业号)

前台/后台执行：fg/bg

切换到后台进程的方式：

  1. 启动进程时，末尾加上&
  2. 进程执行时，先ctrl + z然后bg [job_number]

七、文件系统

挂载

挂载可以把设备文件中的文件系统放入到Linux根目录中的某个位置(挂载点)，然后用户访问挂载点就相当于访问设备中的文件系统。

挂载与卸载命令

  • mount [src] [dst]

  • umount [dst]

查看系统当前挂载的文件

  • mount
  • cat /etc/mtab

文件

文件类型

类型 说明
- 普通文件
d 目录
l 符号链接（软链接）
s 套接字
b 块设备
c 字符设备
p 管道

修改文件权限

  • chown
    • chown [user] [file]：修改文件所有者、组等
  • chmod
    • chmod 777 [file]：设置数字形式的权限
    • chmod a=rwx [file]：设置所有用户（拥有者，同组和其他用户）权限等于rwx

文件备份与恢复

tar

  • tar -cf [dst] [src]：创建备份文件
  • tar -xf [src]：恢复文件

cpio

  • find [src] -print | cpio -o > [dst]：备份
  • cpio -im < [src]：恢复

八、磁盘管理

磁盘基础

磁盘接口的类型有：

  1. IDE
  2. SCSI
  3. Removable Disk

查看磁盘使用情况：

  • df：列出所有文件系统的磁盘使用率
  • du：列出目录的磁盘使用率
  • fdisk [device]：对磁盘进行分区管理（创建、删除、列出分区等等）
    • n：新建
    • q：不保存并退出
    • w：保存并退出
    • t：修改分区类型

磁盘限额

磁盘限额（Disk Quota）是一种系统管理员用来限制文件系统各方面使用率的限制集合。磁盘限额以一种合理的方式来分配有限的磁盘空间。

使用磁盘限额

  1. 重新挂载分区
    • mount -o remount,usrquota,grpquota [device]
  2. 生成quota配置文件
    • quotacheck -ugc [device]
  3. 启动quota
    • quota on [device]
  4. 修改配置项
    • edquota [-u|g] [user|group]

LVM

LVM（Logical volume Manager）是Linux环境下对磁盘分区进行管理的一种机制。它提供了一种比常规的分区模式更加灵活分配空间的方法。

特征：不需要卸载卷就能修改逻辑卷的大小；支持卷快照

结构

LVM大致分为三层：物理卷、逻辑卷组和逻辑卷。

  • 物理卷（PV）：对应一个个硬盘
  • 逻辑卷组（VG）：由多个物理卷组成的空间，可以任用户分配
  • 逻辑卷（LV）：从卷组中划分出来的分区，在逻辑卷之上可以建立文件系统

九、调度任务

Linux提供了一种叫做任务调度（scheduled tasks）的机制，可以自动的执行一些任务。

at

使用

安装并启动服务：yum install at、systemctl start atd。atd表示at的守护进程

然后完成定时任务的创建：at [time] $\longrightarrow$ input task $\longrightarrow$ ctrl + D

如果不想执行则删除任务：atrm [job]

其他

查看并修改时区为东八区：timedatectl和timedatectl set-timezone Asia/Shanghai

cron

和at相比，cron提供了分时天月等时间维度的周期性执行任务的方法。

crontab -e -u [user] ：编辑某个用户的任务表

crontab -l：列出当前用户的定时任务

任务语法

例如：1 * * * * ls /home表示每分钟执行一次ls命令。

分钟 小时 天 月 星期 任务指令

anacron

anacron是对cron的扩展，它对那些非24小时执行的定时任务提供了支持。

可以查看并编辑/etc/anacrontab文件中的内容。
